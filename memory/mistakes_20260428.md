# 2026-04-28 오답노트

## 1. VSCode 첨부 이미지 처리 — reference 즉시 file path 묻기
- **상황**: 세은이 reference 이미지 첨부했지만 VSCode Claude extension이 이미지를 file system에 자동 저장 안 함. 내가 description만으로 톤 매칭 시도하다가 v1~v11까지 11회 실패. 마지막에 file system 검색해서 `c:/Users/orbit/Downloads/ChatGPT Image *.png` 발견.
- **Why**: VSCode 첨부 이미지는 conversation에만 남고 file 자동 저장 없음. description으로 톤을 잡으려 하면 거의 다 빗나감 — 특히 "만화체 vs 미니멀 vs 그 중간" 같은 모호 요청은 reference 없이 매칭 거의 불가능.
- **How to apply**: reference가 화면에 보이면 즉시 "그 이미지 파일 경로 알려달라" 또는 "다운로드 폴더에 저장된 거 있나" 먼저 묻고 시작. description으로 무한 retry 금지. `~/Downloads`, `~/AppData/Local/Temp/claude/...`, `c:/Users/<USER>/AppData/Roaming/Code/...` 순으로 검색해서 직접 찾을 수 있으면 그쪽이 더 빠름.

## 2. "사이즈 줄여" 모호 표현 — 캔버스 vs 캐릭터 비율 먼저 확인
- **상황**: 세은이 "12m 애기 너무 커, 작게 해 줘" 했을 때 strip preview 안에서 height resize로 처리. 세은이 "사진 크기 말고 임마, 애기 자체를 더 어리고 작은 애처럼"으로 정정. 한 번 더 헤맴.
- **Why**: 일러스트 작업에서 "크기"는 (a) 캔버스 내 픽셀 크기, (b) 캐릭터 디자인의 head:body 비율, (c) 옆 캐릭터 대비 상대 크기 셋 중 하나. 모두 다른 처리 필요.
- **How to apply**: "작게" "크게" 같은 모호 표현 들어오면 작업 들어가기 전에 "캐릭터 디자인 자체(head 비율 등)인지, preview에서 보이는 사이즈인지" 한 번 확인. 헤맬 시간보다 짧음.

## 3. 메모리 룰 적용 범위 잘못 일반화 — WJ-Test1 자동 거론 금지

- **Why**: 그 룰의 적용 범위는 정확히 "대시보드/CRM 배포"에 한정. 트위터 자동화는 거기 안 들어감.
- **How to apply**: 메모리 룰 인용 전 description 정확히 읽고 적용 도메인 좁혀서 인용. "비슷한 상황이니까" 식으로 확장 금지. 인용 자체가 misdirection 될 위험 큼.
