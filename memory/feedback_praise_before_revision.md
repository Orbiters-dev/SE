---
name: 수정 요청 DM엔 먼저 칭찬/감사, 그 다음 수정 요청
description: 하서·썸네일·캡션 수정 요청 시 나머지 잘 작성해 준 부분에 대한 감사·칭찬 문구 먼저 반드시 포함
type: feedback
---

인플루언서가 제출한 하서/썸네일/캡션에 수정이 필요할 때, **"잘 작성해 주셔서 감사합니다"** 취지의 칭찬·감사 문구를 먼저 넣고 나서 수정 포인트를 전달한다.

**Why:** 2026-04-21 村上様 DM에서 "ハッシュタグには〜含めていただけますと幸いです"만 넣고 나머지 잘 만든 부분 언급 안 함. 세은 지적: "나머지는 괜찮고 잘 작성해 주셔서 감사하다고도 해야지. 싸가지 밥 말아먹었니?". 수정 요청만 툭 보내면 상대는 전체가 잘못된 줄 오해 + 무례함.

**How to apply:**
- 구조: (인사) → (제출물 공유 감사) → **(잘 만든 부분 칭찬/감사)** → (수정 요청 한 줄) → (마무리)
- 칭찬 문구 예시:
  - 「どちらも素敵にご作成いただき、誠にありがとうございます🙇‍♀️」
  - 「とても丁寧にご作成いただき、ありがとうございます」
  - 「素敵な内容でお作りいただき、感謝申し上げます」
- 수정 요청 앞에 「一点だけ」 「一点のみ」 「一点お願いしたい点がございまして」 등 "이것만"이라는 뉘앙스 붙이면 더 부드러움
- 한국어 번역도 "모두 멋지게 만들어 주셔서 감사합니다" 계열 문장 포함
