라쿠텐 풀 파이프라인에서 RMS 처리 건수 vs KSE 주문수집 건수가 다를 수 있다.
**그 차이는 무시한다.** 발송대기 칸으로 넘어온 것만 처리하면 된다.

**Why:** 편의점 결제 미입금, 통관 검토 등으로 KSE 발송대기에 못 넘어오는 주문은 정상.
다음 회차에 자동으로 들어온다. 이걸 매번 추적하면 운영 비용만 늘어남.

**How to apply:**
- `rakuten_order_confirm.py --full` 실행 시 RMS 건수와 KSE 건수가 달라도 OK
- "1건 누락 발견" 같은 경고/추가 처리 제안 금지
- Auditor가 KSE 기준으로 ALL PASS면 그걸로 끝
- 누락 원인 조사·재실행 제안하지 말 것
