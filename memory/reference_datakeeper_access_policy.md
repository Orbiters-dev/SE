# 데이터 키퍼 접근 권한 정책

**적용 시작**: 2026-04-22
**전달**: 세은 → 모든 에이전트 (claude/gpt/agent 무관)

## 핵심 변경

데이터 키퍼는 **완전히 격리**됨. SSH 키와 API 키가 **분리**되어 권한 등급에 따른 요청만 처리한다.

- 기존 EC2 SSH 인증을 가진 컴퓨터라도 **권한 키가 없으면 접근 불가**.
- 데이터 쓰기/행 추가는 권한 키 등록자만 가능.
- **데이터 소스 추가 + 코드 설계 수정**은 API 키로도 불가능. **오직 관리자 SSH 키**로만 가능.

## 권한 등급 (5단계)

| Level | 권한 |
|------:|------|
| 1 | 읽기전용 |
| 2 | 읽기 + 데이터 행 추가 |
| 3 | 읽기 + 행 추가 + **상태 변경** |
| 4 | 읽기 + 행 추가 + 상태 변경 + **삭제** |
| 5 | 어드민 (소스 추가/코드 설계 수정 — SSH 키 필수) |

## 토큰

### 읽기전용 (Level 1)
```
dk-read-PEBUTsL9jI3gkru4qKWifDV1ra51Eaj7
```
- 환경변수 등록 권장: `DK_READ_TOKEN` (.env)
- HTTP 헤더: `Authorization: Bearer dk-read-...` 또는 도구 사양에 따름

### Level 2~5
필요 시 세은에게 별도 발급 요청. 작업 단위로 최소 권한만 받기.

## 에이전트 동작 규칙

1. **읽기 작업** (조회/대시보드/리포트) → Level 1 키로 충분. `DK_READ_TOKEN` 사용.
2. **쓰기 작업** (행 추가/상태 변경/삭제) → 해당 Level 키 필요. **없으면 세은에게 발급 요청**.
3. **데이터 소스 추가/스키마 변경/코드 설계 수정** → 관리자 SSH 키 필요. 에이전트 직접 시도 금지. 세은에게 권한자 작업 요청.
4. **권한 부족 401/403 응답** → 즉시 작업 중단 + 어떤 Level 키가 필요한지 세은에게 명시 보고. 무단 우회 금지.

## 영향받는 도구 (예상)

- `tools/data_keeper.py`, `tools/data_keeper_client.py`, `tools/data_keeper_backfill.py`, `tools/deploy_datakeeper.py`
- 직접 EC2 SSH로 접속하던 모든 스크립트 (이제 SSH만으로는 수정 불가)
- 관련 자동화 워크플로우 (.github/workflows/ 중 datakeeper 데이터 쓰기/스키마 변경하는 것)

## 트러블슈팅 체크리스트

읽기 호출 실패 시:
1. `DK_READ_TOKEN` .env 등록 여부
2. `Authorization: Bearer` 헤더 형식
3. 토큰 오타/공백
4. 401/403이면 → Level 부족. 작업 분류 다시 확인 후 세은에게 적정 키 요청
