파일을 자동으로 비교·수정(중복 제거, 행 삭제, 병합 등)할 때 **두 파일의 구조가 동일하다고 가정하면 안 된다**. 각 파일의 헤더/스키마를 먼저 실제로 읽어서 다른지 확인한 후 수정해야 한다.

**Why:** 2026-04-20 아마존 0420 오후 주문서 처리에서 오전 파일과 오후 파일의 헤더 컬럼이 영문(`order-id` 등)과 일본어(`注文番号` 등)로 달랐음에도 동일 스키마 가정하고 openpyxl로 행 삭제. 결과적으로 오전의 헤더 문자열(`order-id`)이 am_oids set에 포함되는 등 잘못된 비교 수행. 세은이 "아 진짜 지랄 존나 떠네" / "걍 내가 할 테니까" 지적하며 본인이 처리하겠다고 돌려받음.

**How to apply:**
- 두 파일 비교·병합·중복제거 작업 전에 각 파일의 헤더 행을 먼저 Read/openpyxl로 확인
- 헤더 이름 다르면 **컬럼 매핑 테이블** 먼저 구성 (예: `order-id` ↔ `注文番号`)
- 매핑 없이 index로 비교 금지 — "0번 컬럼" 가정은 다른 템플릿에선 다른 데이터 의미
- 수정 전 **dry-run으로 삭제/변경 대상 행을 먼저 출력**해서 세은에게 보여준 뒤 실행
- 세은에게 돌려 주기 전에 본인이 결과 먼저 확인 (feedback_check_output_yourself.md 연계)
