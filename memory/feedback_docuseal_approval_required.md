계약서 정보(풀네임+이메일) 수령 후 **DOCX/PDF 생성은 자동 OK**. 하지만 **DocuSeal 서명 요청 이메일 발송은 반드시 세은 승인 필수**.

**Why:** 2026-04-22 세은 지시 — "내가 docuseal 이렇게 막 칸 넣지 말라 했잖아.. 왜 나한테 허락도 안 맡고 보내써". 베비몽 계약서 발송 시 자동 11개 서명 필드 배치 + 승인 없이 이메일 발송. 이메일은 회수 불가라 돌이킬 수 없는 실수.

**금지:**
- `send_docuseal_contract.py` 를 `--dry-run` 없이 자동 실행
- 자동 배치된 서명 필드(11개 등) 그대로 발송
- "정보 수령 → 자동 발송" 플로우 따르기 (기존 feedback_contract_info_auto_generate.md는 DOCX 생성까지만 해당)

**반드시:**
- DOCX/PDF 생성은 자동 OK (파일만 저장)
- DocuSeal 발송 전 세은에게 **필드 배치·발송 승인** 명시적으로 받기
- `--dry-run`으로 먼저 확인 → 세은이 OK 하면 실제 발송
- 서명 필드 수·위치 특이 사항 있으면 세은에게 보고

**How to apply:**
1. 계약서 정보 수령 → DOCX/PDF 생성 (자동)
2. 생성 완료 보고 + DocuSeal 발송 **승인 요청** (여기서 멈춤)
3. 세은이 "발송해" 승인하면 실제 발송
4. 발송 완료 후 STEP 6.5 DM 초안 제시
