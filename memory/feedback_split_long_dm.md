type: feedback
---

DM 한 통에 너무 많은 정보/요청을 넣지 말 것.

**예:** STEP 6 (유상 수락 → 상품 선정 + 계약서 정보 수집)을 한 번에 보내지 말고 **분리**:
1. **1차 DM**: 수락 감사 + 【実施内容】【報酬】요약 + 상품 선정 제안 + 링크 + 카라 확인
2. **답장 수신 후 2차 DM**: 【フルネーム】【メールアドレス】 요청

**Why:** 2026-04-22 세은 지시 — "이렇게 너무 길어지면, 일단 상품 선정 먼저 하고 답장 오면 나중에 계약서 발송 위한 정보 물어보면 댐". 인플루언서가 긴 DM을 읽기 어려워하고, 한 번에 답해야 할 정보가 많으면 누락·지연됨.

**How to apply:**
- 상품 선정 + 계약서 정보 동시 요청 금지
- 각 DM은 **하나의 요청만** 하는 구조로 설계
- 순서: 수락 확인 → 상품 선정 → (답장) → 계약서 정보 수집 → (답장) → 계약서 발송 (DocuSign)
