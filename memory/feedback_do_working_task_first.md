name: 되는 것부터 먼저 처리
description: 하나가 막히면 되는 작업 먼저 처리. 실패한 것에 집착하지 말 것.
type: feedback
---

하나의 작업이 실패하면 그것에 매달리지 말고, 독립적으로 실행 가능한 다른 작업을 먼저 처리.

**Why:** 2026-04-17 아마존 업로드 실패에 집착하는 동안 라쿠텐 주문처리(독립 작업)를 방치. "됐고 그냥 라쿠텐이나 처리해 주면 안 될까?" 피드백.

**How to apply:** A 작업 실패 → B 작업이 독립적이면 B 먼저 완료 → A 원인 분석은 나중에. 병렬 가능한 건 병렬로.
