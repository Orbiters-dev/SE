Executor — 9-Stage Analysis Pipeline

Reads DataKeeper → applies ROAS framework → generates proposals per brand.

Pre-flight
DataKeeper Freshness Check
Verify latest data >= T-2. If stale, auto-trigger collection for amazon_ads + search_terms + keywords.
Stage 1-2
Profile & Campaign Data
Load brand config (budget caps, ACOS targets). Fetch all ENABLED campaigns from Amazon Ads API.
Stage 3
Campaign Analysis — ROAS Decision Framework
7 ROAS rules applied per campaign. Generates: pause, reduce_bid, optimize, increase_budget proposals.
ROAS < 1.0 → pause | 1.0-1.5 → bid -30% | 1.5-2.0 → bid -15% 2.0-2.5 → optimize -10% | 2.5-3.0 → optimize | 3.0-5.0 → budget +20% | >5.0 → bid +10%, budget +30%
Stage 4
Search Term & Keyword Data
Primary: DataKeeper (limit=50K). Fallback: direct API. Load execution history for dedup (skip already-executed keywords).
Stage 5
Simulator + DataForSEO + Keyword Analysis
Load backtest JSON → boost negate confidence with SIM data. Cross-reference Google CPC for bid ceiling.
Harvest: ACOS < target, 10+ clicks, 1+ sale → exact match Negate: $0 sales + high spend | ACOS > 2x target | SIM confirmed ($10+) Bids: strong → +20% | inefficient → -20% | no sales → -30%
Stage 6
Keyword Performance Matrix
7d vs 30d trends. Cannibalization detection. Google CPC benchmarking per keyword.
Stage 7-8
Cross-Platform + ASIN P&L
Google/Meta/Shopify context. Full-cost margin per ASIN: COGS + FBA + Export + Returns. Grosmimi 7 product categories.
Stage 9
Save Proposal JSON & Send Email
HTML email with all sections. Budget impact summary. Backtest insights section with SIM badges.
