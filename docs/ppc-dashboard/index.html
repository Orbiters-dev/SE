Executor — 9-Stage Analysis Pipeline

DataKeeper → ROAS framework → proposals per brand. Pre-flight freshness auto-check.

Pre-flight
DataKeeper Freshness Check
Verify latest >= T-2. If stale, auto-collect amazon_ads + search_terms + keywords.
Stage 1-2
Profile & Campaign Data
Brand config (budget, ACOS targets) + fetch all ENABLED campaigns.
Stage 3
Campaign Analysis — ROAS Framework
7 rules per campaign → pause, reduce_bid, optimize, increase_budget.
<1.0→pause | 1.0-1.5→bid-30% | 1.5-2.0→bid-15% | 2.0-2.5→opt-10% | 2.5-3.0→opt | 3.0-5.0→bud+20% | >5.0→bid+10%,bud+30%
Stage 4
Search Term & Keyword Data
DataKeeper (limit=50K) → fallback API. Load execution history for dedup.
Stage 5
Simulator + DataForSEO + Analysis
Backtest JSON → SIM negate boost. Google CPC → bid ceiling. Harvest/negate/bid proposals.
Harvest: ACOS<target, 10+clicks, 1+sale → exact match
Negate: $0sales+high$, ACOS>2x, SIM($10+) | Bids: strong+20%, inefficient-20%
Stage 6
Keyword Performance Matrix
7d vs 30d trends. Cannibalization. Google CPC benchmarks.
Stage 7-8
Cross-Platform + ASIN P&L
Google/Meta/Shopify context. Full-cost margin: COGS+FBA+Export+Returns.
Stage 9
Save JSON & Send Email
HTML email with all sections + budget impact + backtest insights.

Execution — Gmail Reply Trigger

Brand-specific Gmail matching. "Execute all" reply → auto-approve → API calls → confirmation email.

Execution Flow

1. Gmail Search
subject:"[Amazon PPC] {Brand}" from:{approver} — brand-specific match prevents cross-brand execution.
2. Approve All Proposals
Set approved=true on all campaign + keyword proposals. Dedup: check for existing EXECUTED email.
3. API Execution
Campaign: update_campaign_budget(), update_campaign_bid(). Keywords: add_keyword(), add_negative_keyword().
4. Confirmation Email
Subject: "[Amazon PPC] EXECUTED". Changes logged to ppc_executed_{date}.json.
