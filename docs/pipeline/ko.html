⚡ n8n: Syncly Data Processing (64 nodes)
Creator Sync
1. Read Sheet — Google Sheet 1MgEodo... Creator tab
2. Build Lookup Map — Fetch all AT Creators → build username.lower() → recordId map
3. Dedup Check — Match each sheet row username against Lookup Map
    • Exists → UPDATE: refresh Followers, Avg Views, ER, 30-day metrics
    • New → CREATE: Source="Syncly", Status="New", Outreach="Not Started"
    • Blacklist → If sheet Blacklist=TRUE, set Partnership Status="Blacklist"
4. Outreach Type — Recent 30-Day Views > threshold = High Touch, else = Low Touch
    Currently hardcoded 50K in n8n. Daily Config table has "High Touch Req" field (not yet wired)
5. Writeback — AT is source of truth. Sync AT status back to sheet:
    • AT="Blacklist" but sheet not marked → write Blacklist=TRUE to sheet
    • AT="In Progress/Completed" but sheet not updated → write Partnership Status to sheet
    • Uses Write Status to Sheet node to write directly to Google Sheet
Content Sync
1. Read Content Sheet — Content tab, filter only Level=full_matched rows
2. Build Content Lookup — Fetch AT Content table → build postUrl.lower() → recordId map
3. Dedup & Link Creator — Check Post URL against AT for duplicates + resolve username → Creator recordId
    • Exists in AT → UPDATE: refresh Transcript, Caption, Views
    • New → CREATE: Content Status="Pending", link to Creator record
4. Handle Missing Creators — When Content references a Creator not yet in AT:
    • Test mode: Skip (run Creator Sync first)
    • Prod mode: Auto-create Creator with Source="Content Sync (Auto)" → resolve link
5. Resolve Creator Links — Replace _PENDING_ flags with actual Creator recordIds
6. Notify — Send Gmail with sync results: N created, M updated
