Email Reply Config PG 마이그레이션 스펙 (계획)

현재 문제

Reply Handler K99grtW9iWq8V79f의 분류 프롬프트, 응답 템플릿, FAQ가 전부 n8n Code 노드에 하드코딩.
웹 대시보드에서 수정 불가. 브랜드별 커스터마이징 어려움.

해결 방안

Django PG 모델 + REST API 생성.
n8n이 HTTP GET 1회로 config 전부 가져감.
웹 대시보드/Django Admin에서 실시간 수정 가능.

새 Django 모델

onz_email_reply_config — 브랜드별 분류 프롬프트, 템플릿, auto-send 룰
onz_faq_entries — FAQ 지식베이스 (질문, 답변, 키워드, 카테고리)
onz_email_reply_log — 감사 로그 (intent, 자동발송 여부, config 버전)

n8n 변경 단계
1
HTTP GET 노드 추가
GET /api/onzenna/pipeline/email-config/brand/
2
Claude AI 노드 수정
하드코딩 프롬프트 to config.classification.prompt
3
Switch 노드 수정
하드코딩 LT/HT to config.auto_send
4
템플릿 노드 수정
하드코딩 to config.templates[intent]
5
POST reply-log 추가
감사 로그 기록
API 응답 예시

GET /email-config/grosmimi/
{ classification: { prompt, model },
  auto_send: { lt: true, ht: false },
  templates: { accept, faq_gap, normal },
  faq_entries: [...], version: 3 }
