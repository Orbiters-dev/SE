## 전략 분석 실행 방법

### 1. 파이프라인 현황 점검

```bash
# Claude Code에서:
제갈량 소환

# 제갈량이 자동으로:
# 1) 메모리에서 프로젝트 상태 로드
# 2) n8n WF 4개 상태 확인
# 3) content_posts / creators 데이터 현황 확인
# 4) 리스크 매트릭스 도출
```

### 2. 드라이런 테스트

```bash
# Config 설정 (Django API)
POST https://orbitools.orbiters.co.kr/api/onzenna/pipeline/config/2026-MM-DD/
{
  "creators_contacted": 10,
  "test_email_override": "hello@zezebaebae.com"
}

# Draft Gen webhook 트리거
POST https://n8n.orbiters.co.kr/webhook/draft-gen
{"source": "dry_run_test", "batch_size": 10}

# 결과 확인 (n8n API)
GET /api/v1/executions?workflowId=fwwOeLiDLSnR77E1&limit=3
```

### 3. 4중 체크 검증

Draft Gen 실행 후 확인할 항목:

| 체크 | 노드 | 확인 방법 |
|------|------|----------|
| CC1: 브랜드 언급 | Parse Email Draft | crossCheckErrors에 CC1-FAIL 없는지 |
| CC2: Form URL 매칭 | Parse Email Draft | CC2-FAIL 없는지 |
| CC3: 발신자 이름 | Parse Email Draft | CC3-FAIL (William 등) 없는지 |
| CC4: Transcript 필수 | Parse Email Draft | CC4-FAIL = transcript 없는 크리에이터 |
| Cross-Check Gate | Cross-Check Gate | crossCheckPassed=true만 통과 |

### 4. RAG Dedup 검증

```bash
# bulk-check 테스트
POST https://orbitools.orbiters.co.kr/api/onzenna/gmail-rag/bulk-check/
{"emails": ["test@gmail.com", "known_sent@gmail.com"]}

# 기대 응답: 보낸 적 있으면 non-null, 없으면 null
{"results": {"test@gmail.com": null, "known_sent@gmail.com": {...}}}
```

### 5. Creator/Content Sync 수동 실행

n8n UI에서:
1. Workflow: `Orbiters: Syncly Data Processing`
2. "Test workflow" 클릭
3. Creator Sync Trigger 선택 -> 실행
4. Content Sync Trigger 선택 -> 실행

### 6. 테스트 후 정리

```bash
# testEmailOverride 비우기
POST https://orbitools.orbiters.co.kr/api/onzenna/pipeline/config/2026-MM-DD/
{"test_email_override": ""}
```

## 필터 파이프라인 구조

```
Poll (status=Not Started) -> 전체 풀
  -> Extract batch (batchSize from config)
  -> US Only filter           (-N, non-English 제거)
  -> Business Account filter  (-N, 브랜드계정 제거)
  -> Apify Autofill           (이메일 보충)
  -> RAG Email Dedup          (-N, 기발송 제거)
    - sender email 제외 (hello@zezebaebae.com, affiliates@onzenna.com)
    - ManyChat IG handle dedup (graceful skip)
  -> Build Claude Prompt
  -> Claude Generate Draft
  -> Parse Email Draft + CC1-4
  -> Cross-Check Gate (pass/block)
  -> Send Email / Save to Gmail
  -> Update RAG Contact (자동 sync)
```

## 주의사항

1. **testEmailOverride**: 테스트 시 반드시 설정, 끝나면 반드시 비울 것
2. **CC4**: transcript 없는 크리에이터는 무조건 차단 — Content Sync 선행 필수
3. **RAG**: GmailContact 테이블이 비면 dedup 안 됨 — 초기 seed 필요
4. **regex**: n8n 코드에서 `\d` 쓰지 말 것 — JSON roundtrip에서 `\\d`로 변함 — `[0-9]` 사용
5. **config/today/ vs config/{date}/**: today는 GET only, upsert는 date 지정 필요
