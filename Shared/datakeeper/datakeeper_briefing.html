05 — Rules
팀 에이전트 접근 규칙
데이터가 필요할 때 따라야 할 2단계 프로세스.

DO — 이렇게 하세요

  1. Shared/datakeeper/latest/manifest.json 먼저 확인
  2. 채널이 있으면 → 해당 JSON 파일 읽기 (끝)
  3. 채널이 없으면 → API 직접 스크래핑
  4. 스크래핑 후 data_signals/에 YAML 시그널 드롭

DON'T — 절대 하지 마세요

  • ✖ PostgreSQL gk_* 테이블에 직접 쓰기
  • ✖ Shared/datakeeper/latest/ 파일 수정
  • ✖ 이미 수집 중인 채널을 별도 API로 중복 수집

07 — Gotchas
주의사항
데이터 해석 시 반드시 알아야 할 함정들.
Shopify "Amazon" 채널 함정 Critical
shopify_orders_daily의 channel="Amazon"은 진짜 Amazon 판매가 아님
  • FBA MCF = Shopify DTC인데 Amazon 물류 사용 → 실제로 D2C
  • Faire = B2B 도매 → 실제로 B2B
  • 진짜 Amazon = amazon_sales_daily (SP-API)
Grosmimi 가격 변경 Warning
2025-03-01 기준으로 소매가 변경
  • 이전: GROSMIMI_OLD_PRICES 딕셔너리
  • 이후: 현재 Shopify 가격
  • Floor: max(ref, sell) → 음수 할인 방지
Amazon Ads 히스토리 제한 Warning
  • API가 60~90일만 보관
  • Naeiae: 2025-12월부터만 PG에 존재
  • 그 이전 = 물리적으로 불가
데이터 Freshness Info
  • 하루 2회 수집 (PST 0:00, 12:00)
  • /status/ API로 최종 시각 확인
  • 14시간 이내 = fresh, 이후 = stale
