**報酬交渉フロー：**
1. インフルエンサーの提示額を確認
2. ユーザーに報告・相談
3. フォロワー数をもとに逆算して代替金額を提案
4. 再交渉が来たら → 最初の提示額を維持して丁重に断る

**보수 역산 공식：**
> 1. 유저가 제안할 **목표 금액을 먼저 결정** (세은에게 반드시 확인)
> 2. 목표 금액 ÷ 팔로워 수 = 단가 X 역산
> 3. **팔로워 수 × X円 ≤ 목표 금액**이 되는 X 선택 (계산 결과가 목표액을 초과하면 안 됨)
> 4. DM에 **「フォロワー数 × X円」** 형식으로 계산 근거 명시
> 5. 계산 결과를 반올림해서 최종 제안
>
> ⚠️ **1.5円은 고정 기준이 아님. 매번 목표 금액에서 역산할 것.**
> 예: 목표 ~10,000円 / 팔로워 6,000명 → 10,000 ÷ 6,000 ≒ 1.66円 → **1.5円** 선택
> 6,000 × 1.5円 ＝ 9,000円 → **9,000円＋商品1点** 제안 ✅

## 注意事項

- **DM 초안을 유저에게 전달할 때는 반드시 일본어 원문 + 한국어 번역을 함께 제시한다**
- **유저가 직접 작성한 템플릿을 보내주면 그 말투·표현·구조를 그대로 학습하여 이후 DM에 반영한다**
- 返信文は必ず冒頭に「○○様」を入れる（名前不明の場合はユーザーに確認）
- **「○○様」の後、必ず丁寧な挨拶文を入れる（例：「お世話になっております。」「ご返信いただき、誠にありがとうございます。」「ご丁寧にご回答いただきありがとうございます。」等）**
- 報酬金額は必ずユーザーに確認してから提示する
- 軽いトーンのDMには短く返す、フォーマルなDMには丁寧に返す
- 長すぎる文章はインフルエンサーに読まれにくいため注意
- PPSU・漏れ防止・洗浄に関する表現は「核心製品ポイント」の✅/❌ガイドラインを必ず守る
- 競合ブランド（Richell・Pigeon・Combi・Bbox等）の名前を絶対に出さない
- 差別化は製品の特長説明のみで行う。「他にはない機能」などの間接比較表現も避ける
- **タイミングが合わない場合、「投稿楽しみにしています」などビジネスの範囲を超えた表現は使用禁止。こちらから再連絡する旨を伝え、相手に連絡を求めない。**
- **1つのDM内で同じ絵文字を複数回使用しない。**
- **製品選定・推薦のDMを送る際は、必ず該当製品のリンクを本文に添付する。**
- **製品リストを記載する際は、各項目の間に1行空白を入れて見やすくする。**

## CRM 自動更新ルール

**DM 처리 후 반드시 CRM 파이프라인 상태를 업데이트한다.**

### 파이프라인 순서 (대시보드 기준)
```
Not Started → Draft → Sent → Replied → Accepted → Contract → Delivered → Guide → Posted
```

### STEP → CRM 매핑

| STEP | CRM 상태 | 설명 |
|------|----------|------|
| STEP 1 | sent | リチ아ウト DM 발송 |
| STEP 2~5 | replied | 답변·네고·제품추천 |
| 수락 확정 | accepted | 콜라보 수락 |
| STEP 6~6.5 | contract_sent | 계약서 발송 |
| STEP 7 | contract_signed | 서명 완료 + 배송지 수집 |
| STEP 8~10 | sample_shipped | 발송~송장 |
| STEP 10.5 | sample_delivered | 도착 확인 |
| STEP 10.7~11 | guidelines_sent | 가이드라인 발송 + 하서 확인 |
| STEP 12 | posted | 투고 완료 |
| 거절 | declined | 거절 |

### 실행 방법
```bash
python tools/update_crm_status.py --handle {핸들} --step {STEP번호}
python tools/update_crm_status.py --handle {핸들} --status {상태}
```

### 규칙
- **DM 초안 작성 후, CRM 업데이트까지 완료해야 1건 처리 완료**
- 상태는 앞으로만 진행 (뒤로 가면 WARNING)
- declined는 어느 단계에서든 가능
